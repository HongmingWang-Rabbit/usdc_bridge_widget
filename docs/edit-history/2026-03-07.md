## Phase 2: CCTP V2 Parsing Fix

### Problem
`parseCCTPMessage()` used a fixed 116-byte header size (V1 format with 8-byte nonce). CCTP V2 messages have a 148-byte header (32-byte nonce + 8 bytes of finality fields). When parsing V2 messages with the V1 offset, the parser read wallet address bytes as the USDC amount, producing astronomically wrong values.

### Fix
- Read first 4 bytes of message to detect version (`>= 1` = V2)
- V1: 116-byte header (8-byte nonce)
- V2: 148-byte header (32-byte nonce + `minFinalityThreshold` + `finalityLevel`)
- Body offset adjusted accordingly: amount at `headerBytes + 4` (skip body version prefix)
- Test fixed: V2 test now uses proper 32-byte nonce + 2 finality fields

## Phase 3: Recovery Error/Success Feedback

### Problem
When recovery retry completed or failed, there was no user-visible feedback in the widget UI. The banner silently disappeared on success, and errors were only logged to callbacks.

### Changes

#### `src/useRecovery.ts` - Error/success state
- Added `formatRecoveryError(stepName, rawError)`: Maps SDK errors to user-friendly messages
  - "user rejected" / "user denied" → "Transaction was rejected. Please try again."
  - "timed out" → "Recovery timed out. Please try again."
  - `Simulation failed on X` → "Transaction simulation failed on X. The transfer may have already been claimed."
  - Fallback: "Recovery failed at {step} step. Please try again."
- Added `lastError: { bridgeId: string; message: string } | null` — per-bridge error scoping
- Added `lastSuccess: string | null` — success message with 8-second auto-clear
- Added `successTimerRef` for proper timer cleanup on unmount
- Replaced RegExp-based prefix stripping with `startsWith()` + `slice()` (avoids regex injection from SDK step names)

#### `src/components/RecoveryBanner.tsx` - UI feedback
- Success banner: Green background with CheckIcon when `lastSuccess` is set and no pending bridges remain
- Per-bridge error: Shows error message only on the banner matching `lastError.bridgeId`
- Error hidden during active recovery (prevents stale error flicker)

#### `src/components/PendingItemCard.tsx` - Clipboard fix
- Added rejection handler to `navigator.clipboard.writeText()` (graceful on non-HTTPS / permission denied)
